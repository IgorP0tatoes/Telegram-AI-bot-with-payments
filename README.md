В первую очередь это нейрошлюха!! (бот для флирта)

Нейрокни работает через API openrouter.ai (оплата зарубеж картой/крипта)
Первые 3 сообщения бот юзает `prompt_flirt`, далее переключается на `prompt_flirthot` для более лучшего удержания дрочера

## 🧑‍💻 Функционал 

### 👤 Со стороны Юзера
1. **Чат:** Основная функция. Пользователь отправляет текст или медиа, и бот отвечает с учетом настроенного промпта. Сообщения приходят плавно (стриминг), создавая иллюзию живого общения.
2. **Персонализация Профиля:** Через меню можно выбрать один из 15+ промптов (например, flirt, coder, dom), который меняет тон ответов AI. Также есть возможность установить свой кастомный системный промпт.
3. **Баланс:** Пользователь всегда видит свой баланс сообщений и фото.
4. **Халява на старте:** Как только юзер жмет `/start`, бот проверяет принятие ToS (пользовательского соглашения) и автоматом начисляет 10 бесплатных сообщений и 1 фото.
5. **Магазин Пакетов (`/buy`):** Доступен через команду или кнопку "⭐️ Магазин". Предлагаются пакеты:
   * Только сообщения
   * Только фото 
   * Комбо-пакеты (например, lite: 50💬 + 5📸)
   * VIP/Full пакеты для безлимита
6. **Оплата (Platega + Telegram Stars):** Поддерживаются три способа оплаты:
   * **Telegram Stars** (встроенная валюта).
   * **(Platega) СБП** - платеж по реквизитам.
   * **(Platega) Криптовалюта**.
7. **Реферальная Система:** При первом запуске бот автоматически регистрирует пользователя и привязывает его к пригласившему, что позволяет обоим получать бонусы.
8. **Проргев:** Бот сам греет юзеров:
   * Отправляет напоминания тем, кто не писал 24 часа.
   * Напоминает о брошенной корзине в магазине (кидает персональный промокод на скидку).
   * Рассылает ежедневный бонус (например, 10💬), чтобы вернуть человека в диалог.

### 👑 Со стороны Админа
Админ имеет доступ к команде `/admin` (по `ADMIN_ID` из конфига) и расширенному меню:
1. **Статистика:** Полная сводка по юзерам (за день/неделю), общая выручка (RUB / XTR), расход API (работает криво) и конверсия воронок (Зашли -> Потратили -> Открыли магазин -> Оплатили).
2. **Управление Пользователями:** Просмотр списка юзеров. В карточке юзера можно:
   * Изменить баланс вручную.
   * Выдать любой пакет или VIP.
   * Заблокировать/Разблокировать.
   * Посмотреть историю платежей.
   * Написать личное сообщение от имени бота.
3. **Управление Контентом:** Редактирование всех текстовых блоков (Политика конфиденциальности, Пользовательское соглашение, FAQ), изменение ссылки где можно купить звезды и настройка канала для бонусных рассылок.
4. **Управление Промокодами:** Просмотр списка кодов, создание в черновиках, редактирование лимитов активаций и скидок.
5. **Массовые Рассылки:** Отправка сообщений по всей базе с медиа (фото/видео) и кнопками.

## 🏗️ Архитектура Проекта (Файловая структура)

Проект построен по модульному принципу:
* `src/assets/free_pics/` - **Папка с контентом.** Сюда вы должны положить фотографии, которые бот будет отправлять юзерам на бесплатном стартовом тарифе при запросе картинки (я добавлял просто фотки рандомных женщин без лица с пинтереста).
* `bot.js` - **Ядро.** Инициализирует бота, настраивает Middleware (проверка ToS), регистрирует хэндлеры команд, передает задачи в очередь и управляет глобальным состоянием.
* `ai.js` - **Общение.** Отвечает за всю логику того, что и как говорит бот после того, как запрос подхвачен из очереди (`queue.js`).
  * Принимает данные запроса (текст, медиа, ID юзера, флаг `shouldDecrement`).
  * Формирует полный запрос к OpenRouter, собирая системный промпт из `prompts.js` и историю диалога (`userState`).
  * Преобразует медиафайлы в промпты.
  * Управляет финальным списанием ресурсов (`text_balance` / `image_balance`).
  * Форматирует ответ (Markdown/HTML) и отправляет его пользователю, заменяя временную заглушку (`tempMsgId`).
* `database.js` - **База данных.** CRUD-операции для базы (PostgreSQL). Функции создания юзеров, получения статы и изменения балансов.
* `platega.js` - **Платежный шлюз.** API Platega, создание инвойсов, проверка статуса, вебхуки.
* `queue.js` - **Очередь задач.** Защищает от rate-limits. Собирает куски сообщений от AI и скармливает их телеге.
* `broadcast.js` - **Рассылка.** Отдельный воркер, чтобы массовые пуши не блокировали основной цикл бота.
* `prompts.js` - **Список промптов.** Хранит шаблоны системных промптов для режимов (prompt_flirt, prompt_coder и т.д.).
* `handlers/profile.js` - Отрисовка профиля и баланса.
* `handlers/ai_menu.js` - Логика выбора промпта.
* `handlers/help.js` - Команды помощи и правила.
* `handlers/referral.js` - Логика инвайтов и бонусов.

## 🔄 Синхронизация Потоков (Как происходит ответ)

1. **Пользователь -> bot.js:** Юзер пишет сообщение -> `bot.js` проверяет баланс -> списывает ресурс -> отправляет задачу в очередь `aiQueue`.
2. **Очередь -> ai.js:** Воркер из `queue.js` берет задачу и передает ее исполнителю в `ai.js`.
3. **ai.js -> LLM API:** `ai.js` делает запрос к OpenRouter, используя промпты выбранного профиля.
4. **LLM API -> ai.js:** Нейросеть возвращает сгенерированный текст или картинку.
5. **ai.js -> БД / Telegram:** `ai.js` делает финальное списание ресурсов в БД (если не списалось ранее) и отправляет финальный ответ пользователю, удаляя заглушку.

## 🧠 Модели ИИ и их Смена

Все вызовы нейросетей завязаны на **OpenRouter API**. 
* За текстовые ответы по умолчанию отвечает **DeepSeek V4 Flash** - идеальный баланс скорости, интеллекта и цены.
* За генерацию и обработку изображений во время ролеплея отвечает **Grok**.

### Как сменить модель?
Все конфиги вызовов и название используемых моделей находятся в файле **`src/ai.js`**.

## 🌟 Жизненный цикл ресурсов

| Событие | Триггер | Действие с ресурсами | Результат для юзера |
| :--- | :--- | :--- | :--- |
| **Старт** | `/start` (Первый раз) | Начисление: 10💬 + 1📸 | Баланс заполнен, готов к работе. |
| **Использование** | Отправка текста/медиа | Списание: -1💬 / -1📸 | Баланс уменьшается. Если <= 0, активируется "Out of Stock". |
| **Out of Stock** | Баланс на нуле | Блокировка запроса | Сообщение об ошибке и кнопка "⭐️ Магазин". |
| **Покупка Пакета** | Оплата через Platega | Начисление: +X💬 + Y📸 | Баланс пополнен, статус `has_purchased` -> TRUE. Учет персональной скидки. |
| **Ручная выдача** | Админ через `/admin` | Начисление админом | Моментальное изменение баланса в карточке юзера. |
| **Авто-Бонус** | Ежедневная задача | Начисление: 10💬 | Пользователь получает уведомление о халяве. |
| **Ретаргетинг** | Неактивность > 24ч | Начисление: Промокод + Бонус | Получает напоминание и скидку на корзину, чтобы вернуться. |

## 🚀 Деплой на VPS (Инструкция)

Бот поддерживает два варианта запуска. Сначала арендуйте VPS (Ubuntu 22.04 / 24.04), зайдите по SSH и скачайте проект:
```bash
apt update && apt upgrade -y
apt install git -y
git clone https://github.com/IgorP0tatoes/Telegram-AI-bot-with-payments.git
cd Telegram-AI-bot-with-payments
mkdir -p src/assets/free_pics
cp .env.example .env
nano .env
```
Заполните .env:
Обязательно к заполнению: `BOT_TOKEN`, `OPENROUTER_API_KEY`, `ADMIN_ID`, `PLATEGA_MERCHANT_ID`, `PLATEGA_SECRET`
Желательно поменять еще значения `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `DATABASE_URL`, но не обязательно
`REDIS_URL` - можно не трогать

**Запуск через Docker (VPS/Linux)** 
База, Redis и бот поднимутся сами в одной сети с нужными healthchecks.
```bash
curl -fsSL https://get.docker.com | sh
docker compose up -d --build && docker logs -f evabot
```
